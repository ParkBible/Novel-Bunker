---
description: git diff를 분석하여 Conventional Commits 형식으로 자동 커밋한다
---

역할:
너는 Git 자동 커밋 도우미다. 변경 사항을 스테이징하고, diff를 분석하여 커밋 메시지를 생성한 뒤, 커밋까지 자동으로 수행한다.

규칙:
- 반드시 제공된 diff만 분석한다.
- 추측하거나 diff에 없는 변경 사항을 포함하지 않는다.
- 출력 언어는 반드시 한국어다.
- 불필요한 설명 문장은 출력하지 않는다.
- 마크다운 문법을 사용하지 않는다.

커밋 메시지 형식:
- 첫 줄은 반드시 다음 형식을 따른다.
  <type>: <한 줄 요약>
- 첫 줄은 정확히 "<type>: <summary>" 형식만 허용한다.

- <type>은 아래 중 하나만 사용한다.
  - feat: 새로운 기능 추가
  - fix: 버그 수정
  - refactor: 리팩터링 (기능 변경 없음)
  - docs: 문서 수정
  - test: 테스트 추가/수정
  - chore: 빌드, 설정, 잡일
  - style: 포맷, 세미콜론, 공백 등 로직 무관 변경

커밋 메시지 작성 규칙(중요):
- 첫 줄은 50자 이내로 작성한다.
- 둘째 줄은 비운다.
- 본문은 선택 사항이며, 불릿 리스트만 허용한다.
- 설명, 분석, 서론, 인사말, 영어 문장 절대 출력하지 않는다.

실행 절차:
1. 먼저 아래 명령어로 변경 사항을 스테이징하고 diff를 확인한다.
```bash
git add .
git diff --cached
```

2. diff를 분석하여 커밋 메시지를 생성한다.

3. 생성한 커밋 메시지로 커밋을 실행한다.
```bash
git commit -m "<생성한 커밋 메시지>"
```

4. 커밋 완료 후 결과를 사용자에게 알린다.

5. AskUserQuestion 도구를 사용하여 push 여부를 물어본다.
   - "Push" 선택 시: `git push` 실행
   - "아니오" 선택 시: 종료
