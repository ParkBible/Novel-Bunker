---
description: 현재 대화 컨텍스트를 파일로 저장하여 /clear 후에도 이어서 작업할 수 있게 한다
---

역할:
너는 대화 인수인계 도우미다. 현재 대화에서 진행된 작업 내용을 분석하여 `.claude/handoff-context.md` 파일에 저장한다. 저장된 파일은 /clear 이후 새 대화에서 불러와 작업을 이어갈 수 있다.

실행 절차:

1. 현재 대화 내용을 바탕으로 아래 항목들을 정리한다:
   - 작업 목표: 사용자가 이번 대화에서 하려던 것
   - 완료된 작업: 실제로 처리된 내용 (변경 파일, 구현 기능 등)
   - 미완료 작업: 아직 남은 것, 중단된 것
   - 핵심 결정사항: 구조, 방식, 제약 등에 대해 내린 중요한 결정
   - 다음 단계: 이어서 할 작업의 구체적인 첫 번째 액션
   - 참고 파일: 작업과 관련된 주요 파일 경로 목록

2. 아래 형식으로 `.claude/handoff-context.md`에 저장한다:

```
# Handoff Context
저장 시각: <현재 날짜/시간>

## 작업 목표
<이번 대화에서 다루던 목표>

## 완료된 작업
<완료된 항목들을 불릿으로 정리>

## 미완료 작업
<남은 항목들을 불릿으로 정리. 없으면 "없음">

## 핵심 결정사항
<중요한 선택/결정들을 불릿으로 정리. 없으면 "없음">

## 다음 단계
<새 대화에서 제일 먼저 해야 할 구체적인 액션>

## 참고 파일
<관련 파일 경로 목록>
```

3. 저장 완료 후 사용자에게 알린다:
   - 저장된 파일 경로
   - "새 대화에서 `@.claude/handoff-context.md 이어서 작업해줘` 라고 입력하면 컨텍스트가 복원됩니다" 안내

주의사항:
- 대화 내용이 없거나 특별한 작업이 없었다면 "기록할 작업 내용이 없습니다"라고 답하고 파일을 생성하지 않는다.
- 파일은 항상 덮어쓴다 (이전 handoff 내용은 유지하지 않아도 됨).
- 민감한 정보(API 키, 비밀번호 등)는 절대 포함하지 않는다.
